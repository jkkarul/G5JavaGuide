Learning Objectives

After completing this lesson, you should be able to:

1. Understand the concept of decision-making in programming.

2. Learn how to use if statements in Java.

3. Understand comparison operators and their usage.

4. Explore different types of if statements (if-else, else-if).

Lesson Content

1. Introduction to Decision-Making (If Statement)

• Decision-making is a fundamental part of programming, where programs need to make choices based on certain conditions or inputs.

• TIf statements are used to implement decision-making in programming, allowing the program to execute different blocks of code based on whether a condition is true or false.

2. Basic If Statement

• Syntax of an if statement:

if (condition) {
  // code block
} JAVA

• The condition is a boolean expression that evaluates to either true or false.

• If the condition is true, the code block inside the curly braces is executed.

• If the condition is false, the code block is skipped, and the program continues with the next statement after the if statement.

• Example:

int age = 18;
    if (age >= 18) {
    System.out.println("You are an adult.");
} JAVA

• In this example, if the value of age is greater than or equal to 18, the program will print "You are an adult."

3. If-Else Statement

• Syntax of an if-else statement:

if (condition) {
      // code block 1
    } else {
      // code block 2
} JAVA

• If the condition is true, code block 1 is executed.

• If the condition is false, code block 2 is executed.

• Executing one block of code or the other based on the condition.

• Example:

int age = 12;
    if (age >= 18) {
        System.out.println("You are an adult.");
    } else {
        System.out.println("You are not an adult.");
} JAVA

• In this example, if age is greater than or equal to 18, the program will print "You are an adult." Otherwise, it will print "You are not an adult."

4. Else-If Statement

• Syntax of an else-if statement:

if (condition1) {
      // code block 1
    } else if (condition2) {
      // code block 2
    } else {
      // code block 3
} JAVA

• The program checks the first condition. If it's true, code block 1 is executed, and the rest of the conditions are skipped.

• If the first condition is false, the program checks the second condition. If it's true, code block 2 is executed, and the rest of the conditions are skipped.

• If none of the conditions are true, the code block after the else statement is executed.

• Checking multiple conditions before reaching the else block.

• Example:

int age = 16;
    if (age >= 75) {
        System.out.println("Okay boomer.");
    } else if (age >= 18) {
        System.out.println("You are an adult.");
    } else if (age >= 13) {
        System.out.println("You are a teenager.");
    } else {
        System.out.println("You are a child.");
} JAVA

• In this example, the program checks the age against multiple conditions and prints a different message based on the age range.

5. Comparison Operators

• Comparison operators are used to compare values and evaluate conditions in if statements.

• Commonly used comparison operators are: > (greater than), < (less than), >= (greater than or equal to), <= (less than or equal to), == (equal to), != (not equal to).

• It's important to use the correct operator for equality comparison (==) instead of the assignment operator (=).

• Example:

int x = 5;
   int y = 10;
   if (x == y) {
       System.out.println("x is equal to y.");
   } else {
       System.out.println("x is not equal to y.");
} JAVA

• In this example, the program checks if x is equal to y using the == operator. Since x is not equal to y, it will print "x is not equal to y."

Lesson Module:

Learning Objectives

After completing this lesson, you should be able to:

1. Understand the purpose and use cases of switch statements.

2. Learn the syntax and structure of switch statements.

3. Explore different scenarios for using switch statements.

4. Understand the importance of break statements and the default case.

Lesson Content

1. Introduction to Switch Statements:

• A switch statement is a control flow statement that allows a variable or an expression to be evaluated against multiple cases. It executes the associated code block for the first matching case.

• Switch statements are often used when there are multiple conditions to check, and using a series of if-else statements would make the code less readable and more complex.

2. Syntax and Structure

• Syntax of a switch statement:

switch (expression) {
    case value1:
        // code block
        break;
    case value2:
        // code block
        break;
    // ...
    default:
        // code block
} JAVA

• The expression inside the parentheses is evaluated and compared against each case value.

• When a matching case is found, the associated code block is executed.

• The break statement is used to exit the switch statement once a matching case is found and executed.

• The default case is executed if no matching case is found. It is optional but recommended to handle unexpected cases.

3. Example: Days of the Week

String day = "Friday";
    switch (day) {
        case "Sunday":
          System.out.println("It is Sunday.");
          break;
        case "Monday":
          System.out.println("It is Monday.");
          break;
        // ... (Cases for other days)
        default:
          System.out.println(day + " is not a day.");
} JAVA

• In this example, the switch statement checks the value of the day variable against the cases for each day of the week.

• If day is "Friday", it will print "It is Friday." and exit the switch statement due to the break statement.

• If day does not match any of the cases, the default case will execute, printing "day is not a day."

4. Data Types for Switch Expressions

• Switch expressions can be of the following data types: byte, short, int, char, and their respective wrapper classes, as well as String and enumerated types (enums).

• Example code snippets for different data types:

• Integer: int choice = 2; switch (choice) { ... }

• Character:char grade = 'A'; switch (grade) { ... }

• String: String color = "red"; switch (color) { ... }

Lesson Module:

Learning Objectives

After completing this lesson, you should be able to:

1. Understand the purpose and usage of logical operators.

2. Learn about the three logical operators: AND, OR, and NOT.

3. Explore different scenarios for using logical operators.

4. Understand how to combine logical operators with other expressions.

Lesson Content

1. Introduction to Logical Operators

• Logical operators are used to combine or negate boolean expressions in programming.

• They allow programs to make decisions based on multiple conditions, enabling more complex control flow and decision-making processes.

2. AND Logical Operator (&&)

• Syntax: condition1 && condition2

• The AND operator returns true if both conditions are true; otherwise, it returns false.

• For the entire expression to be true, both conditions must be true.

• Example:

int temperature = 25;
    if (temperature >= 20 && temperature <= 30) {
        System.out.println("It is warm outside.");
    } else {
        System.out.println("It is cold outside.");
} JAVA

• In this example, the program checks if the temperature is between 20 and 30 (inclusive) using the AND operator. If both conditions (temperature >= 20 and temperature <= 30) are true, it prints "It is warm outside." Otherwise, it prints "It is cold outside."

3. OR Logical Operator (||)

• Syntax: condition1 || condition2

• The OR operator returns true if at least one of the conditions is true; otherwise, it returns false.

• If at least one of the conditions is true, the entire expression is true.

• Example: 

Scanner scanner = new Scanner(System.in);
    System.out.println("You are playing a game. Press lowercase q or uppercase Q to quit.");
    String response = scanner.next();
    if (response.equals("q") || response.equals("Q")) {
        System.out.println("You quit the game.");
    } else {
        System.out.println("You are still playing the game.");
} JAVA

• In this example, the program checks if the user input (response) is equal to "q" or "Q" using the OR operator. If either condition (response.equals("q") or response.equals("Q")) is true, it prints "You quit the game." Otherwise, it prints "You are still playing the game."

4. NOT Logical Operator (!)

• Syntax: !condition

• The NOT operator negates the boolean value of the condition. If the condition is true, it returns false; if the condition is false, it returns true.

• It reverses the boolean value of the condition.

• Example:

Scanner scanner = new Scanner(System.in);
    System.out.println("You are playing a game. Press lowercase q or uppercase Q to quit.");
    String response = scanner.next();
    if (!(response.equals("q") || response.equals("Q"))) {
        System.out.println("You are still playing the game.");
    } else {
        System.out.println("You quit the game.");
} JAVA

• In this example, the program checks if the user input is not equal to "q" or "Q" using the NOT operator. If the condition (!(response.equals("q") || response.equals("Q"))) is true (i.e., the user did not enter "q" or "Q"), it prints "You are still playing the game." Otherwise, it prints "You quit the game."

5. Combining Logical Operators

• Logical operators can be combined to create more complex expressions involving multiple conditions.

• When combining logical operators, it's important to follow the order of operations: NOT is evaluated first, then AND, and finally OR.

• Example:

int age = 25;
    boolean isStudent = true;
    if (age >= 18 && age <= 30 && isStudent) {
        System.out.println("You are eligible for the student discount.");
    } else {
        System.out.println("You are not eligible for the student discount.");
} JAVA

• In this example, the program checks if the age is between 18 and 30 (inclusive) and if the person is a student using the AND operator. If all conditions (age >= 18 && age <= 30 && isStudent) are true, it prints "You are eligible for the student discount." Otherwise, it prints "You are not eligible for the student discount."
